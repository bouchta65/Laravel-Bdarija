Migration Example

        use Illuminate\Database\Schema\Blueprint;
        use Illuminate\Support\Facades\Schema;
        
        class CreatePostsTable extends Migration
        {
            public function up()
            {
                // ha bhalach tssaweb wahd table smito posts 
                Schema::create('posts', function (Blueprint $table) {
                    $table->id();
                    $table->string('title');
                    $table->text('content');
                    $table->foreignId('user_id')->constrained(); // Foreign key dyal l user
                    $table->timestamps(); // Kayzid timestamps (created_at, updated_at)
                });
            }
        
            public function down()
            {
                Schema::dropIfExists('posts'); // hadi l function li katkhali table ytmssah mn database
            }
        }

Eloquent Models

Eloquent huwa ORM (Object-Relational Mapping) dyal Laravel. Kul table f database 3ndo model li kaykhdem m3ah.

Basic Model


        namespace App\Models;
        
        use Illuminate\Database\Eloquent\Model;
        
        class Post extends Model
        {
            protected $fillable = ['title', 'content', 'user_id']; // katdir l'attributes li ghat7ot fihom data
        
            // Relationship m3a User model
            public function user()
            {
                return $this->belongsTo(User::class); // Each post kay7tajo user
            }
        
            // Relationship m3a Comments
            public function comments()
            {
                return $this->hasMany(Comment::class); // Post 3ando bzaf dyal comments
            }
        
            // Custom scope
            public function scopePublished($query)
            {
                return $query->where('status', 'published'); // Kayjib lik posts li status dyalhom published
            }
        }

        // lmohim kaynin bzaf dyal l methods dyal Relationship hawwel t9aleb 3lihom o 3rafhom

// app/Http/Controllers/AuthController.php
public function login(Request $request)
{
    $credentials = $request->validate([
        'email' => 'required|email',
        'password' => 'required'
    ]);

    if (Auth::attempt($credentials)) {
        $request->session()->regenerate();
        return redirect()->intended('dashboard');
    }

    return back()->withErrors([
        'email' => 'Credentials ghala·π≠.',

LARAVEL: A Comprehensive Summary

1. Understanding Routes

Routes determine how your application responds to web requests

Basic Route

Route::get('/posts', [PostController::class, 'index']);
  • ‚Üí When someone visits yoursite.com/posts, this route handles it
  • ‚Üí It calls the index method in PostController

Route with Parameters

  • ‚Üí {id} is a dynamic value - could be /posts/1, /posts/2, etc.
  • ‚Üí The value gets passed to the show method automatically

2. Working with Controllers

Controllers organize your code and handle requests

Basic Controller Method

  • ‚Üí Post::all() gets all posts from the database
  • ‚Üí view('posts.index') loads the template from resources/views/posts/index.blade.php
  • ‚Üí compact('posts') makes $posts available in the view

public function store(Request $request) { $validated = $request->validate([ // Validate input 'title' => 'required|max:255', 'content' => 'required' ]); Post::create($validated); // Create post return redirect()->route('posts.index'); // Redirect back }
  • ‚Üí validate() checks if the submitted data is valid
  • ‚Üí Post::create() saves the new post to the database
  • ‚Üí redirect()->route() sends the user back to the posts list
